## Findings considered and rejected

- text_fragments RLS "null user_id bypass": policy logic actually *hides* NULL
  rows from users; works as designed for service-role cache writes.
- Gemini stream byte-cap "not enforced": the stream loop `return`s immediately
  after `safeClose()`; cap works.
- `.single()` crash on admin role check: supabase-js returns errors rather than
  throwing; check fails closed.
- JSON.parse uncaught in phrases route: parse is inside the model-fallback
  try/catch.
- GeminiAdapter mic-track leak: stream lifecycle is owned by
  `useSharedMicStream`/`useVoiceRecorder`, which stop tracks.
- streak `yesterdayStr` UTC slice: only misfires at server TZ offsets beyond
  ±12h; servers run UTC. Hardening folded into the timezone finding (#4).
- admin-seed prompt injection via client `sounds`: admin-only, schema-validated,
  fails closed — marginal.
- Gemini key rotation/versioning governance: ops concern, not a code finding.
- `dayOfYear` off-by-one in daily-plan rotation: cosmetic variance only.
- Adding generic memoization to every Client Component: rejected because the
  measured cost is mounting and shipping inactive feature trees, not evidence
  of widespread expensive pure renders.
- Compressing the four existing PNG files: deferred because they are confined
  to auth presentation and are already served through `next/image`; route
  bundle and query work has higher leverage.
